Getting Acquainted with Git and GitHub

January 21, 2016

The most important concepts we learned this week were version control and how Git and GitHub works, all super handy tools. Version control keeps track of your files and all the changes you've made and when you made them. So if you screw up, you can just go back to an earlier version of the file before things started going downhill. These different saved versions of your file are called a commit; you can think of them as a snapshot in time of your code. A good analogy, would be a checkpoint in a video game, if you die or fail in the middle of a video game, you can just go back to your last save point or in our case the last commit and proceed from there.
