Positiong with CSS

February 5, 2016

This week we began tinkering with CSS; the most difficult and by far the frustrating aspect for me of CSS is trying to understand how positioning works so I'm going to try to explain it here in this blog post. There are four different types of position values: static, fixed, relative, and absolute. Let's start off with the easiest position to understand which would be the default position of static. These elements simply stack in a standard one-after-another order the same way they are ordered in your code. So whatever comes first in your code will be displayed first closer to the top of the page and so on so forth.

Elements given a fixed position value are completely independent of everythign else on the web page and will always be positioned based on the browser viewport. It's often used to create a floating element that stays in the same position on the page no matter how far down you scroll. You can see in the pictures below that box "One" given a position: fixed stays in the same place, even after scrolling from the top to the bottom of the page.

Relative positioning allows you to move an element without disrupting the normal flow of a page. The element moves relative to its default static positioning. Using the properties of top, right, bottom, and left we can position the element from its default position. In the image, below we have four boxes that are displayed-inline. I changed box "Two" to be position: relative and adjusted top: 20px and left: 20px from its original position. Notice that the other elements are still displayed as if "Two" was still in its original place.

In contrast to relative positioning, absolute positioning takes the element out of normal flow which allows us to move that element anywhere on the page. An element that is positioned absoltuely does not take up any space when placing other elements, almost as if the element doesn't exist. In the image below, I gave box "Two" the same top: 20px and left: 20px as in the example above, but changed position: relative to position: absolute.

As you can see, box "Two" now overlaps with other elements on the rest of the page and box "Three" and "Four" now shift over to the left on the inline block as if box "Two" never existed. The position of aboslute elements is determined by it parent elements. If all the parent elements are static, or in this case: there are none, then the element is positioned based on the body of the page. Hopefully, you found this post useful in getting introduced to positiong with CSS.
